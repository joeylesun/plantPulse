# SETUP.md — PlantDoc

Detailed installation and usage guide. If you're a grader trying to reproduce results, start here.

## Prerequisites

- **Python 3.10 or newer** (tested on 3.10 and 3.11; 3.12 works but Chroma/torch wheels are slower to land)
- **~10 GB free disk** (PlantVillage is ~1 GB, model checkpoints + ChromaDB add another ~300 MB)
- **GPU strongly recommended for training** (CPU training will take ~10x longer). Inference runs fine on CPU.
- **An OpenAI API key** for the RAG pipeline (uses `gpt-4o-mini`, which is cheap — budget a few cents for all of the evaluations)

## 1. Clone and install

```bash
git clone <your-repo-url>
cd plantdoc
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
```

If you run into `torch` installation errors, install it first from the official instructions at https://pytorch.org/get-started/locally/ (the right wheel depends on your CUDA version), then rerun the `pip install -r requirements.txt`.

## 2. Download the PlantVillage dataset

Needed only for training. If you're just running the app with a pretrained checkpoint, skip to step 4.

The PlantVillage dataset is publicly available on Kaggle:
- https://www.kaggle.com/datasets/abdallahalidev/plantvillage-dataset

Download the "color" subset (~1 GB). After extracting, your directory should look like:

```
data/plantvillage/color/
├── Apple___Apple_scab/
├── Apple___Black_rot/
├── Apple___Cedar_apple_rust/
├── ...
└── Tomato___healthy/
```

38 subdirectories total, each containing images of that plant+disease combination.

## 3. Configure environment variables

```bash
cp .env.example .env
```

Edit `.env` and fill in your keys:

```
OPENAI_API_KEY=sk-...            # required for RAG
MODEL_PATH=models/plantdoc_resnet50.pt
CHROMA_DIR=models/chroma_db
```

## 4. Train the vision model

Open the training notebook:

```bash
jupyter lab notebooks/02_training.ipynb
```

Run all cells. This will:
- Build DataLoaders with the 5-technique augmentation pipeline
- Compute a random baseline for reference (~2.6%)
- Fine-tune ResNet-50 for 10 epochs
- Save the best checkpoint to `models/plantdoc_resnet50.pt`
- Save test-set predictions to `models/test_predictions.npz` (used by the error-analysis notebook)
- Plot training curves to `docs/training_curves.png`

**Expected runtime**: ~1.5 hours on a consumer GPU (RTX 3080/4070), ~10+ hours on CPU.

**If you don't want to train from scratch**, download the pretrained checkpoint from `[add your Drive/Box link here after uploading]` and place it at `models/plantdoc_resnet50.pt`.

## 5. Build the RAG knowledge base

```bash
python -m src.build_knowledge_base
```

This embeds the 38 disease documents in `data/disease_knowledge_base.json` using a local sentence-transformer (no internet/API call needed for embeddings), and persists a ChromaDB under `models/chroma_db/`. Takes ~30 seconds.

You should see output like:

```
Loading knowledge base from data/disease_knowledge_base.json...
Loaded 38 disease documents.
Building vector store at models/chroma_db/...
Done. Vector store contains 38 embeddings.
```

## 6. Run the app

```bash
streamlit run app.py
```

Opens at http://localhost:8501. Upload any leaf photo (try one from the test set, or a real photo from your garden) and you should see:

1. The original image and a Grad-CAM heatmap side by side
2. Top-1 prediction with confidence, plus a collapsible top-3 list
3. Treatment advice generated by the RAG pipeline
4. A chat box for follow-up questions about the diagnosis

## 7. (Optional) Reproduce the ablation and error analysis

```bash
jupyter lab notebooks/03_ablation_study.ipynb    # frozen vs finetuned, aug vs no aug
jupyter lab notebooks/04_error_analysis.ipynb    # confusion matrix, worst classes, hardest examples
jupyter lab notebooks/05_gradcam_visualization.ipynb
jupyter lab notebooks/06_rag_evaluation.ipynb    # RAG vs vanilla LLM
```

Each notebook is standalone given a trained checkpoint at `models/plantdoc_resnet50.pt` and saved predictions at `models/test_predictions.npz` (both produced by notebook 02).

## Common issues

**`FileNotFoundError: data/plantvillage/color` when training.** The dataset isn't in the expected path. See step 2.

**`FileNotFoundError: models/plantdoc_resnet50.pt` when running the app.** You haven't trained a model yet. Run notebook 02, or download the pretrained checkpoint.

**`FileNotFoundError: models/chroma_db` when using RAG.** Run `python -m src.build_knowledge_base`.

**`openai.AuthenticationError` when asking a question.** Your `OPENAI_API_KEY` isn't set, or `.env` isn't being read. Confirm with `python -c "from dotenv import load_dotenv; import os; load_dotenv(); print(os.getenv('OPENAI_API_KEY')[:6])"`.

**CUDA out of memory during training.** Reduce `batch_size` in `notebooks/02_training.ipynb` from 32 to 16 or 8.

**`langchain_chroma` ImportError.** Chroma and LangChain split their packages across multiple repos; make sure you installed `langchain-chroma` specifically (it's in `requirements.txt`).
